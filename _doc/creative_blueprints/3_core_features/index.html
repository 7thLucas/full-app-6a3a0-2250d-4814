Core Features

Three screens. One clean flow. Real transactions.

User (v1) Merchant (future) Admin (future)
🏠

Home Screen

Card-based service grid — UPI Payment (active), Recharge, Bills, Services, Travel, More (placeholders). Tap UPI to begin.

💸

Payment Screen

Amount input + "Pay Now" button. Initializes Razorpay, converts to paise (×100), handles callbacks. Disposes cleanly on exit.

✅

Result Screen

Centered UI — Success with Payment ID, or Failed status. No extra buttons, no extra logic. Clean and final.

⚡

Razorpay Integration

razorpay_flutter package. PAYMENT_SUCCESS and PAYMENT_ERROR listeners. Placeholder Key ID — swap for production key at launch.

🏗

Pluggable Architecture

screens / services / widgets / models — clean UI/logic separation. Each service is a pluggable module. Add taxi, food, courier without redesigning core.

📱

Play Store Ready

Flutter project structure follows Play Store conventions. Minimal design, no animation overload, fast performance from day one.
